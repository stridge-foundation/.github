# Security Policy

Stridge takes the security of our software and our users' funds seriously.

## Reporting a vulnerability

If you believe you've found a security vulnerability in any Stridge project,
please report it privately — **do not open a public issue or PR**.

Use [GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
on the affected repository (the **Security → Report a vulnerability** tab).

We'll acknowledge your report, keep you updated as we investigate, and credit
you once the issue is resolved — unless you'd prefer to remain anonymous.

## Learn more

See the [Stridge security model](https://docs.stridge.com/security) for how the
gateway protects funds end to end: non-custodial routing, IP allowlists,
browser-safe gateway keys, and split, policy-checked signing keys.
